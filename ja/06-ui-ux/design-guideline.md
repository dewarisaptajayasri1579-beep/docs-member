# デザインガイドライン — 中央会員&SSOハブ

## 1. ドキュメントの目的

このドキュメントは、中央会員&SSOハブの開発におけるデザインガイドラインとユーザーエクスペリエンス (UX) ガイドラインを提供するものです。

---

# 2. デザインの原則

1. **モバイルファースト**: デザインはモバイルデバイスに優先され、次にタブレットとデスクトップに適応されます。
2. **プレミアム&ミニマル**: クリーンで過剰な要素がなく、各要素には目的があります。
3. **一貫性**: コンポーネント、スペーシング、カラー、フォントは全体的に一貫しています。
4. **アクセシビリティ**: カラーコントラストはWCAG 2.1 AAの基準を満たします。
5. **ステートに応じたレスポンシブ**: 各ステート (ローディング、エラー、成功、空) に応じた表示が必要です。

---

# 3. テーマ

システムは2つのテーマをサポートしています。

- **ダークモード** (デフォルト)
- **ライトモード**

ユーザーはプロフィールページまたは設定ページでテーマを切り替えることができます。テーマの設定はローカルに保存されます。

## 4.3 グラデーション

ボタンの主な色とハイライト要素はグラデーションを使用します。

```css
background: linear-gradient(135deg, #7C3AED, #A78BFA);
```

# 8. メインコンポーネント

## 8.1 ボタン

| バリエーション | 表示 | 使用 |
| --- | --- | --- |
| プライマリ | グラデーション紫、白テキスト | メインアクション (ログイン、有効化、支払い) |
| セカンダリ | 紫のボーダー、紫のテキスト | セカンドアクション (キャンセル、戻る) |
| デンジャー | ピンクの背景、赤のテキスト | 削除、キャンセル |
| ゴースト | 透明、ミュートテキスト | アクションリンク |

- デフォルトサイズ: `height: 48px`、`border-radius: 12px`、`font-weight: 600`
- モバイル用にフルウィドス
- ステート: `default`、`hover`、`active`、`disabled`、`loading`

## 8.2 インプットフィールド

- `height: 52px`
- `border-radius: 12px`
- `border: 1px solid border`
- ラベルは常に上に配置されます (フローティングラベルはありません)
- ステート: `default`、`focus`、`error`、`disabled`
- エラーメッセージは入力下に表示され、赤色になります
