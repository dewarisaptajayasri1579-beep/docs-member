# Application Overview — Central Membership & SSO Hub

## 1. System Name

**Central Membership & SSO Hub**

Tagline:

> **One Account, All Applications.**

## 2. System Summary

Central Membership & SSO Hub is a centralized system that functions as:

- **Identity Hub**: One account to access all `SaaS` products within the ecosystem.
- **License Hub**: Manages licenses for each product per user.
- **Billing Hub**: Handles package selection and subscription payments.
- **SSO Provider**: Issues authentication tokens (`JWT`) accepted by each `SaaS` application.

## 3. Background

Each `SaaS` product in the ecosystem has different functions. Without a centralized identity system, users would have to register, manage passwords, and pay separately for each application.

Central Membership Hub is here to simplify this experience with:

- one-time registration for all products,
- one dashboard to view all subscriptions,
- one login process (`SSO`) to access all applications,
- and one place to manage payments.

## 5. Key Concepts

### 5.1 Member Account

A Member Account is the primary user identity within the ecosystem.

One Member Account can have:

- multiple different product subscriptions,
- each with a unique `License-ID`,
- and different package statuses per product.

### 5.2 License-ID

A `License-ID` is a unique code generated by the system each time a user activates a subscription for a product.

Format:

```
[PREFIX_PRODUK]-[XXXX]-[XXXX]-[XXXX]
```

Example for NOTO:

```
NTO-A1B2-C3D4-E5F6
```

`License-ID`:

- is unique per user per product,
- displayed on the member dashboard,
- sent to email,
- and used for support reference.

### 5.3 Subscription Packages (Per Product)

Each product has its own list of packages.

Available package types:

- **Free**: Active without payment. No expiration date.
- **Trial**: Full access for a limited period.
- **Paid**: Monthly or annual subscription.

Currently, **NOTO** offers a **Free Forever** package with full feature access.

### 5.4 Single Sign-On (SSO)

Users only need to log in once to the Membership Hub to access all applications with active licenses.

The `SSO` mechanism uses `JWT` (JSON Web Token) issued by the Hub and verified by each application.

### 5.5 Grace Period

When a paid subscription ends and the user has not renewed, the system provides a **Grace Period** before access is suspended.

During the `Grace Period`:

- users can still access the application,
- the system displays urgent notifications to renew,
- and user data remains secure.

After the `Grace Period` ends without renewal, access is suspended.

## 6. Payment Gateway

The system supports two `payment gateway`s:

- **Midtrans**
- **Xendit**

Users can choose from the available payment methods during checkout.

## 7. Main Modules

Main modules of the system:

1. Account Registration & Verification
2. Member Profile Management
3. Product & Package Catalog
4. Checkout & Payment
5. License Management
6. `SSO` / `OAuth2` Server
7. Notifications & Email
8. Super Admin Dashboard
9. Audit Log

## 8. Development Principles

- Financial data and user identities must be protected with the highest security standards.
- Each `SaaS` application must not store user password data (delegated to the Hub).
- Payment processes must not be partially saved in case of failure.
- The system must be able to add new products without altering the core architecture.
- `License-ID` must be verifiable by applications independently using `JWT`.
- The `Grace Period` must be clearly communicated to users before access is suspended.
