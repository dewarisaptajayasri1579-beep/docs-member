# ビリング プラン — 中央会員 & SSO ハブ

## 1. ドキュメントの目的

このドキュメントでは、SaaS エコシステム内の各製品のビリング プランの構造、管理ルール、および製品ごとの特定の条件を説明します。

## 2. プランの原則

1. 各SaaS製品は独自のプランを定義します。
2. 1つの会員は異なる製品の異なるプランを購入できます。
3. **Free / 無料**プランには支払いが必要ではなく、有効期限はありません。
4. **有料**プランには有効期限があり、有効期限が過ぎると自動的に終了します。
5. プランの価格と機能はスーパーアドミンによって管理されます。

---

# 3. 製品: NOTO

## 3.1 プランのステータス

現在、NOTOでは1つのプランを提供しています。

| プラン | 価格 | 有効期限 | ステータス |
|---|---|---|---|
| Free Forever | IDR 0 | 無制限 | 有効 |

## 3.2 NOTOのFree Foreverプランの条件

- 有効期限はありません。
- 全てのNOTO機能にフルアクセスできます (Early Access)。
- 有限のクォータ (財務アカウントの数、財務ポジション、トランザクション) はありません。
- 登録から最初の日から利用可能です。

> **注記**: 将来、有料のNOTOプランが導入されると、機能またはクォータの制限が別のドキュメントで定義されます。

# 4. 有料製品のプラン構造 (テンプレート)

有料製品がエコシステムに追加されたとき、プランは以下の構造を採用します。

## 4.1 標準プランの種類

| プラン | 目的 |
|---|---|
| Free / Trial | 新規ユーザーが試用するために |
| Starter | 個人ユーザーまたは基本的なニーズのユーザー |
| Pro | 活発なユーザーと中程度のニーズのユーザー |
| Business | 小規模チームまたは進んだニーズのユーザー |

## 4.2 支払いサイクル

| サイクル | 説明 |
|---|---|
| 月次 | 月ごとに支払われる |
| 年次 | 年ごとに支払われる (通常は安く) |

## 4.3 プラン構造データの例

```json
{
  "product_code": "APP",
  "plan_name": "Pro",
  "billing_cycle": "monthly",
  "price": 99000,
  "currency": "IDR",
  "duration_days": 30,
  "features": [
    "機能A",
    "機能B",
    "機能C"
  ],
  "is_active": true
}
```

# 5. ライセンスの比較 (ステータスに基づく)

| ライセンス ステータス | アプリケーション アクセス | 説明 |
|---|---|---|
| `active` | はい、フル | ライセンスが有効で有効期限が過ぎていない |
| `grace_period` | はい、警告あり | ライセンスが有効期限を過ぎており、トレーニング期間中 |
| `suspended` | いいえ | トレーニング期間が終了し、アクセスがブロックされる |
| `cancelled` | いいえ | メンバーまたは管理者によってライセンスがキャンセルされた |
| `free` | はい、フル | 無料プラン (NOTO の場合) |

# 6. トレーニング期間

## 6.1 トレーニング期間の条件

- トレーニング期間は **有料** プランのみに適用されます。
- **無料** プランにはトレーニング期間はありません。
- トレーニング期間の長さは、システムの設定によって製品ごとに定義されます。
- デフォルトのトレーニング期間は **7 日** です。

## 6.2 トレーニング期間の通知スケジュール

| 時間 | 通知 |
|---|---|
| 有効期限 7 日前 | メール: ご利用の契約が即座に終了します |
| 有効期限 3 日前 | メール: ご契約を即座に更新してください |
| 有効期限 1 日前 | メール: ご契約が終了します |
| トレーニング期間開始時 (有効期限) | メール: ご契約が終了し、7 日間で更新できます |
| トレーニング期間中 3 日目 | メール: 4 日以内にアクセスがブロックされる |
| トレーニング期間中 6 日目 | メール: 明日、アクセスがブロックされます |
| トレーニング期間終了時 (有効期限) | メール: アクセスがブロックされました |

## 6.3 トレーニング期間中の更新

- メンバーはいつでもトレーニング期間中で更新できます。
- 更新が成功すると、元の有効期限から計算される `expired_at` が次のようになります。
  ```
  expired_at_baru = expired_at_lama + durasi_paket
  ```

---

# 7. プランの更新 (アップグレード / ダウングレード)

- **アップグレード**: メンバーはいつでも上位のプランにアップグレードできます。残りの有効期間は比例的に計算されます。
- **ダウングレード**: メンバーは有効な支払いサイクルの終了時に下位のプランにダウングレードできます。

> アップグレード/ダウングレードのプロレート計算ルールは、詳細なビジネスルールドキュメントで定義されます。

---

# 8. ビリング プランの受容基準

- NOTO の無料プランは支払いなしで有効になります。
- 有料プランは支払いが確定された後のみ有効になります。
- トレーニング期間が正常に実行されます。
- メール通知が予定どおり送信されます。
- ライセンスが停止されたときにデータは削除されません。
- メンバーのデータはライセンスが停止されたときも削除されません。
