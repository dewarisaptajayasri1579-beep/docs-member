# License and Subscription Management — Central Membership & SSO Hub

## 1. Document Purpose

This document explains how the system manages member licenses and subscriptions, including License-ID generation, license lifecycle, and the display of license information to members.

## 2. License-ID Concept

### 2.1 Definition

A License-ID is a unique code generated by the system for each active subscription of a member to a specific product.

### 2.2 Format

```
[PREFIX_PRODUK]-[XXXX]-[XXXX]-[XXXX]
```

- **PREFIX_PRODUK**: Short SaaS product code (2–4 characters, uppercase).
- **[XXXX]**: 4-character alphanumeric uppercase segment, randomly generated.

**Example:**
```
NTO-A1B2-C3D4-E5F6   → NOTO license belonging to a member
APP-X9Y8-Z7W6-V5U4   → Application B license belonging to the same member
```

### 2.3 License-ID Properties

- Globally unique across the entire system.
- Cannot be changed after generation.
- Valid for only one member and one product.
- Non-transferable.
- Displayed on the member dashboard and sent via email.

# 3. License Lifecycle

```
Activated
    │
    ▼
[active]
    │
    ├── (Free Plan)
    │       Never expires → remains [active/free] forever
    │
    └── (Paid Plan)
            │
            ▼ (expired_at reached)
        [grace_period]
            │
            ├── Member renews → returns to [active]
            │
            └── Grace Period ends
                    │
                    ▼
                [suspended]
                    │
                    ├── Member pays → returns to [active]
                    │
                    └── Member does not return → data remains stored
```

## 3.1 License Status

| Status | Kode | Description |
|---|---|---|
| Active | `active` | Active license, full access |
| Free Forever | `active_free` | Free plan, no expiration |
| Grace Period | `grace_period` | Expired, still within grace period |
| Suspended | `suspended` | Grace Period ended, access blocked |
| Cancelled | `cancelled` | Cancelled by member or admin |

---

# 4. License-ID Generation

### 4.1 When License-ID is Generated

- When a member activates a free plan for the first time.
- When the first payment for a specific product is successful.

### 4.2 Generation Rules

- The system checks License-ID uniqueness before saving.
- If a member already has a license for the same product (renewal), the existing License-ID is **not replaced**.
- A License-ID is only regenerated if the previous license was `cancelled` and the member repurchases.

For paid plans, additional information:

| Information | Example |
|---|---|

| Event | Email |
|---|---|
| New license activation | Activation confirmation + License-ID |
| Payment successful | Invoice + renewal confirmation |
| D-7 until expiration | Expiration reminder |
| D-3 until expiration | Urgent reminder |
| D-1 until expiration | Last day reminder |

# 8. License Rules

1.  A member can only have one active license per product at a time.
2.  License-ID is unique and non-transferable.
3.  Free licenses do not have an expiration date.
4.  Grace Period only applies to paid licenses.
5.  Member data is not deleted even if the license is suspended.
6.  License renewals for expired licenses are calculated from the old expiration date, not the payment date.
7.  Cancelled licenses cannot be restored; a new purchase will generate a new License-ID.

---

# 9. Acceptance Criteria

-   License-ID is formatted `[PREFIX_PRODUK]-[XXXX]-[XXXX]-[XXXX]` and unique across the system.
-   License-ID is displayed on the dashboard after activation.
-   Confirmation email with License-ID is sent after activation.
-   License status changes automatically according to conditions (active → grace_period → suspended).
-   Email notifications are sent according to schedule.
-   Members can view all their licenses in one dashboard.
-   Members cannot access the application if the license is suspended.
